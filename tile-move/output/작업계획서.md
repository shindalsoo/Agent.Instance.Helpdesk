# 작업계획서

## 프로젝트 개요
- **프로젝트명**: 2048 Web Edition  
- **목표**: 브라우저에서 다운로드나 설치 없이 바로 플레이할 수 있는 간단한 2048 게임 구현  
- **핵심 기능**:
  - 4x4 격자 표시  
  - 화살표 키(또는 WASD) 입력으로 블록 이동  
  - 같은 숫자의 블록이 합쳐질 때 점수 증가  
  - 새 블록 생성 로직 (값은 2 또는 4)  
  - 게임 종료/재시작 기능  

## 기술 스택
| 영역 | 도구 |
|------|------|
| 프론트엔드 UI | **HTML5** (마크업), **CSS3** (스타일링, Flexbox/Grid), **Vanilla JavaScript** (ES6+) |
| 빌드/배포 | 없음(단순 파일) – `index.html`을 직접 열면 실행 가능 |
| 테스트 | 브라우저 콘솔 + 단위 테스트 필요 시 Jest (옵션) |

## 파일 구조
```
project-root/
├─ index.html          # 메인 HTML 페이지
├─ src/
│  ├─ app.js           # 게임 로직 및 이벤트 처리
│  ├─ board.js         # 격자 상태 관리, 렌더링
│  ├─ utils.js         # 유틸리티(랜덤 블록 생성 등)
│  └─ style.css        # UI 스타일
└─ README.md           # 프로젝트 설명
```

> **주의**: 모든 소스 파일은 반드시 `src/` 폴더 아래에 배치합니다.

## 기능 명세

| # | 기능 | 상세 내용 |
|---|------|-----------|
| 1 | 초기 화면 | 로고, 스코어 표시, 시작 버튼(또는 바로 게임 시작) |
| 2 | 격자 렌더링 | 4x4 그리드, 각 셀은 `div.cell` 클래스로 표현 |
| 3 | 블록 생성 | 게임 시작 시 두 개의 랜덤 위치에 2 또는 4 생성<br>이동 후 새 블록 하나 추가 |
| 4 | 입력 처리 | 화살표 키(↑↓←→) 및 WASD 지원<br>키 이벤트를 캡처해 이동 방향 결정 |
| 5 | 움직임 로직 | 행/열 단위로 슬라이드 & 병합<br>한 번의 이동에서 같은 숫자는 한 번만 합쳐짐 |
| 6 | 점수 계산 | 블록이 합쳐질 때마다 그 값(합된 수)을 스코어에 추가 |
| 7 | 게임 종료 판정 | 더 이상 이동 가능한 셀이 없으면 "Game Over" 메시지 표시 |
| 8 | 재시작 기능 | "Restart" 버튼을 누르면 초기화 및 새 게임 시작 |
| 9 | UI/UX 개선 | 블록 색상(숫자별), 애니메이션(옵션) |
|10 | 반응형 디자인 | 모바일에서도 터치 입력 지원(옵션) |

## 구현 우선순위
1. **프로젝트 초기화**  
   - `index.html`, `src/style.css`, `src/app.js` 생성  
2. **격자 렌더링** (`board.js`)  
3. **블록 생성 로직** (`utils.js`)  
4. **키 입력 처리** (`app.js`)  
5. **움직임 & 병합 알고리즘** (`board.js`)  
6. **점수 계산 및 UI 업데이트** (`app.js` + `style.css`)  
7. **게임 종료 판정** (`board.js`)  
8. **재시작 기능** (`app.js`)  
9. **UI/UX 개선** (색상, 애니메이션)  
10. **반응형 & 터치 입력 지원** (선택)

> 각 단계는 단일 파일(또는 모듈)에서 완결하도록 설계합니다.

## 개발 절차
1. `index.html`에 기본 구조와 CSS 링크 삽입  
2. `src/board.js`에서 격자 상태(`grid`)를 2D 배열로 관리하고 렌더링 함수 구현  
3. `src/utils.js`에 `getRandomEmptyCell`, `addRandomTile` 등 유틸리티 작성  
4. `src/app.js`에서 이벤트 리스너 등록, 입력 매핑 및 게임 루프 연결  
5. 테스트를 통해 이동/합치기 로직 검증  
6. UI 개선 및 최종 배포(모든 파일을 하나의 폴더에 두고 브라우저에서 바로 열어 테스트)

> **Tip**: 모듈화를 위해 ES6 `export`/`import` 사용, 브라우저 호환성을 위해 최신 스크립트 태그(`type="module"`)를 활용하세요.
