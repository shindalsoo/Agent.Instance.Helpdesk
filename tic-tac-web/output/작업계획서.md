# 프로젝트 개요
**Tic‑Tac‑Toe (XO) Web Game**

- **목표**: 사용자가 별도의 다운로드 없이 브라우저에서 바로 즐길 수 있는 간단한 3x3 Tic‑Tac‑Toe 게임 구현  
- **특징**:
  - 2인 동시 플레이(로컬, 동일 장치)
  - 승리/무승부 판정
  - 재시작 버튼
  - 최소 UI (버튼 또는 CSS 그리드)

---

## 기술 스택

| 영역 | 선택 이유 |
|------|----------|
| **HTML5** | 구조와 게임판 생성 |
| **CSS3** | 깔끔한 레이아웃과 반응형 디자인 |
| **Vanilla JavaScript (ES6+)** | 외부 라이브러리 없이 구현 가능, 학습용으로 적합 |

> **파일 저장 경로**: 모든 소스 파일은 `src/` 폴더 아래에 배치합니다.

## 기능 명세

| 기능 | 상세 내용 |
|------|----------|
| **1. 화면 초기화** | `<div id="board">` 안에 3x3 그리드 생성(각 셀은 버튼 혹은 div). `index.html`에서 기본 구조를 정의하고, `styles.css`로 스타일링. |
| **2. 사용자 입력 처리** | 각 셀이 클릭되면 JS가 이벤트 리스너(`click`)를 통해 현재 플레이어(X/O)를 판단 후 해당 셀에 표시. 이미 선택된 셀은 무시. |
| **3. 승리/무승부 판정** | 매 차례마다 `checkWinner()` 함수 호출: 8개의 패턴(행, 열, 대각선) 중 하나가 같은 기호를 가졌는지 검사. 결과에 따라 알림(`alert` 또는 모달) 표시. |
| **4. 게임 재시작** | “Reset” 버튼을 눌러 board와 상태 변수 초기화. 모든 셀의 텍스트/클래스 제거. |
| **5. UI 피드백** | 현재 플레이어를 화면에 표시(`div#turn`). 승리 시 해당 플레이어 색상 강조. 무승부 시 “Draw” 표시. |

## 구현 우선순위

1. **기본 구조 만들기** (`index.html`, `styles.css`)  
2. **게임판 생성 및 클릭 이벤트 연결** (`app.js`)  
3. **플레이어 전환 로직** (X → O)  
4. **승리/무승부 판정 함수** 구현  
5. **재시작 기능** 추가  
6. **UI 개선(색상, 반응형)**

> **Tip**: 단계별로 `console.log`를 활용해 디버깅하고, 테스트 케이스를 직접 실행하며 확인합니다.

## 개발 가이드라인

- **코드 품질**: 함수 단위 분리(`initBoard`, `handleClick`, `checkWinner`, `resetGame`)  
- **주석**: 주요 로직 앞에 한 줄 주석으로 설명 삽입  
- **변수명**: 의미가 명확하도록 (`currentPlayer`, `boardState` 등)  
- **스타일링**: CSS 변수(`--primary-color`) 사용해 색상 관리
