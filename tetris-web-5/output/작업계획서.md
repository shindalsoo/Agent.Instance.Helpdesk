# 작업계획서: Web-based Tetris Game 개발

## 1. 프로젝트 개요
본 프로젝트는 HTML5, CSS3, 그리고 Vanilla JavaScript를 사용하여 웹 브라우저에서 실행 가능한 클래식 테트리스 게임을 구현하는 것을 목표로 합니다. 별도의 외부 라이브러리 없이 순수 자바스크립트만으로 게임 엔진, 충돌 로직, 점수 시스템을 구축하여 가볍고 빠른 실행 환경을 제공합니다.

## 2. 기술 스택
- **Language**: HTML5, CSS3, JavaScript (ES6+)
- **Rendering**: HTML5 Canvas API (게임 보드 렌더링용)
- **Development Environment**: Browser (Chrome/Edge/Safari 등 최신 브라우저)

## 3. 파일 구조
모든 소스 코드는 `src/` 폴더 아래에 위치합니다.

```text
project-root/
│
├── index.html              # 게임 실행 메인 페이지 및 UI 구조
├── src/
│   ├── style.css           # 게임 스타일링 (레이아웃, 애니메이션, UI 디자인)
│   ├── game.js             # 핵심 게임 엔진 (루프, 상태 관리, 입력 처리)
│   ├── tetrominoes.js      # 테트리미노 모양 및 색상 데이터 정의
│   └── utils.js            # 충돌 검사, 그리드 생성 등 유틸리티 함수
└── README.md               # 프로젝트 설명서
```

## 4. 기능 명세

### 4.1 핵심 게임 엔진 (Core Engine)
- **Grid System**: 10x20 크기의 2차원 배열로 게임 보드를 관리합니다.
- **Game Loop**: `requestAnimationFrame`을 사용하여 일정한 프레임 속도로 게임 상태를 업데이트하고 화면을 다시 그립니다.
- **Gravity/Tick**: 일정 시간 간격으로 현재 조작 중인 테트리미노를 아래로 이동시킵니다.

### 4.2 테트리미노 제어 (Tetromino Control)
- **Shapes**: I, J, L, O, S, T, Z 7가지 모양의 블록 구현.
- **Movement**: 좌/우 이동, 하단 가속(Soft Drop).
- **Rotation**: 블록을 90도 회전시키는 로직 (벽 충돌 시 회전 제한 처리 포함).
- **Collision Detection**: 벽면, 바닥, 그리고 이미 쌓인 블록과의 충돌을 감지하여 이동/회전을 제한합니다.

### 4.3 게임 규칙 및 시스템 (Game Rules)
- **Line Clearing**: 한 줄이 가득 차면 해당 줄을 제거하고 상단의 블록들을 아래로 내립니다.
- **Scoring System**: 제거된 라인 수에 따라 점수를 부여합니다 (1줄 < 2줄 < 4줄 순으로 가중치 증가).
- **Level Progression**: 특정 점수 도달 시 게임 속도(Gravity)를 증가시켜 난이도를 높입니다.
- **Game Over**: 새로운 블록이 생성될 공간이 없을 경우 게임을 종료하고 결과 화면을 표시합니다.

### 4.4 사용자 인터페이스 (UI/UX)
- **Scoreboard**: 현재 점수, 레벨, 제거된 라인 수를 실시간으로 표시합니다.
- **Next Piece Preview**: 다음에 나타날 블록을 미리 보여주어 전략적 플레이를 지원합니다.
- **Control Instructions**: 키보드 조작법(화살표 키, 스페이스바 등) 안내를 제공합니다.
- **Start/Restart UI**: 게임 시작 및 재시작 버튼을 구현합니다.

## 5. 구현 우선순위
1.  **Phase 1: 기초 환경 구축** (HTML Canvas 설정, 기본 그리드 생성)
2.  **Phase 2: 테트리미노 데이터 및 렌더링** (블록 모양 정의 및 화면 출력)
3.  **Phase 3: 이동 및 충돌 로직 구현** (좌우/하단 이동, 회전, 벽 충돌 검사)
4.  **Phase 4: 게임 규칙 완성** (라인 클리어, 점수 계산, 게임 오버 로직)
5.  **Phase 5: UI/UX 고도화** (Next Piece 표시, 애니메이션 효과, 스타일링 완성)
6.  **Phase 6: QA 및 최적화** (버그 수정 및 코드 리팩토링)
