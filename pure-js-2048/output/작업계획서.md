# 2048 퍼즐 게임 개발 작업계획서

## 1. 프로젝트 개요
본 프로젝트는 사용자가 짧은 시간 동안 직관적으로 즐길 수 있는 순수 JavaScript(Vanilla JS) 기반의 **2048 퍼즐 게임** 구현을 목표로 합니다. 별도의 프레임워크 없이 브라우저 표준 기술만을 사용하여 가볍고 빠른 실행 성능을 제공하며, 모바일과 데스크톱 모두에서 부드러운 조작감을 제공하는 것을 핵심 가치로 합니다.

## 2. 기술 스택
- **Language:** JavaScript (ES6+)
- **Markup:** HTML5
- **Styling:** CSS3 (Flexbox, Grid, Transitions)
- **Environment:** Browser (Chrome, Safari, Edge 등 최신 브라우저)

## 3. 파일 구조
모든 소스 코드는 `src/` 디렉토리 아래에 관리합니다.

```text
project_root/
│
├── index.html              # 게임 실행 메인 페이지
├── output/
│   └── 작업계획서.md        # 현재 문서
└── src/
    ├── index.html          # (참고용) 구조적 HTML
    ├── style.css           # 게임 스타일 및 애니메이션 정의
    ├── script.js           # 핵심 게임 로직 및 이벤트 핸들링
    └── constants.js        # 게임 설정 값 (그리드 크기, 목표 점수 등)
```

## 4. 기능 명세

### 4.1 게임 엔진 (Core Logic)
- **Grid Management**: 4x4 크기의 2차원 배열로 게임판 상태를 관리합니다.
- **Movement & Merging**:
    - 상(Up), 하(Down), 좌(Left), 우(Right) 4방향 이동 로직을 구현합니다.
    - 동일한 숫자를 가진 블록이 충돌할 경우 하나의 블록으로 합쳐지며 값이 2배가 됩니다.
    - 한 번의 이동 단계에서 동일한 블록이 중복해서 합쳐지지 않도록 처리합니다 (예: [2, 2, 2, 2] -> [4, 4]).
- **Tile Spawning**: 이동이 발생할 때마다 빈 칸에 2 또는 4 값을 가진 새로운 타일을 무작위로 생성합니다.

### 4.2 게임 규칙 및 상태 (Game Rules)
- **Score System**: 블록이 합쳐질 때마다 합쳐진 숫자의 크기만큼 점수를 획득합니다.
- **Win Condition**: 타일의 숫자가 2048에 도달하면 승리 메시지를 출력합니다.
- **Lose Condition**: 더 이상 이동할 수 있는 칸이 없고, 인접한 숫자가 같은 블록이 없을 경우 게임 종료(Game Over) 처리를 합니다.

### 4.3 사용자 인터페이스 (UI/UX)
- **Visual Feedback**: 타일이 이동하거나 합쳐질 때 CSS `transition`을 사용하여 부드러운 움직임을 구현합니다.
- **Responsive Design**: 데스크톱 화면뿐만 아니라 모바일 기기에서도 적절한 크기로 보이도록 반응형 레이아웃을 적용합니다.
- **Game Info**: 현재 점수(Score)와 최고 점수(Best Score)를 상단에 실시간으로 표시합니다.
- **Controls**: 
    - 데스크톱: 키보드 방향키(Arrow Keys) 지원.
    - 모바일: 스와이프(Swipe) 제스처 인식 기능 포함.

## 5. 구현 우선순위
1. **[P0] 핵심 로직 구현**: 4x4 그리드 생성, 타일 이동 및 병합 알고리즘 완성.
2. **[P1] 게임 상태 관리**: 점수 계산, 승리/패배 판정, 새로운 타일 생성 로직.
3. **[P2] UI 레이아웃 구성**: HTML 구조 잡기 및 기본 CSS 스타일링 (Grid 적용).
4. **[P3] 애니메이션 및 인터랙션**: CSS Transition을 이용한 부드러운 움직임, 키보드/터치 이벤트 연결.
5. **[P4] 폴리싱**: 최고 점수 저장(LocalStorage), 게임 재시작 기능, UI 디자인 디테일 수정.
