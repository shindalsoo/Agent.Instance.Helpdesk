# 작업계획서: 2048 퍼즐 게임 개발 (Pure JavaScript)

## 1. 프로젝트 개요
본 프로젝트는 별도의 프레임워크 없이 순수 JavaScript(Vanilla JS)를 사용하여 구현된 **2048 퍼즐 게임**입니다. 사용자가 상하좌우 방향키나 스와이프 동작을 통해 숫자 블록을 이동시켜 동일한 숫자를 합치고, 최종적으로 2048 숫자를 만드는 것을 목표로 합니다. 가볍고 빠른 반응성, 직관적인 UI/UX를 최우선으로 합니다.

## 2. 기술 스택
- **Language:** HTML5, CSS3, JavaScript (ES6+)
- **Rendering:** DOM Manipulation (for simplicity and performance in a small grid)
- **Animation:** CSS Transitions & Keyframes
- **Environment:** Browser (Chrome, Safari, Firefox, Edge 등 최신 브라우저)

## 3. 파일 구조
모든 소스 코드는 `src/` 디렉토리 아래에 관리됩니다.

```text
project_root/
│
├── index.html              # 게임 실행 메인 페이지
├── output/
│   └── 작업계획서.md        # 현재 문서
└── src/
    ├── index.html          # HTML 구조 정의
    ├── css/
    │   └── style.css       # 게임 레이아웃 및 애니메이션 스타일
    └── js/

        ├── engine.js       # 2048 핵심 로직 (Grid, Move, Merge)
        └── ui.js           # DOM 업데이트 및 이벤트 리스너 관리
```

## 4. 기능 명세

### 4.1 핵심 게임 엔진 (`engine.js`)
- **Grid Management**: 4x4 크기의 2차원 배열로 그리드 상태를 관리합니다.
- **Tile Generation**: 게임 시작 시와 이동 후 빈 칸에 2 또는 4 숫자를 가진 타일을 랜덤하게 생성합니다.
- **Movement Logic**:
    - 상(Up), 하(Down), 좌(Left), 우(Right) 네 가지 방향의 이동 로직을 구현합니다.
    - 같은 숫자의 타일이 충돌할 경우 하나의 타일로 병합(Merge)됩니다.
    - 한 번의 이동에서 동일한 타일이 두 번 이상 병합되지 않도록 처리합니다.
- **Game State Detection**:
    - **Win Condition**: 2048 타일 생성 시 승리 판정.
    - **Lose Condition**: 더 이상 이동할 수 있는 빈 칸이나 병합 가능한 타일이 없을 때 패배 판정.

### 4.2 UI/UX 및 애니메이션 (`ui.js`, `style.css`)
- **Score Display**: 현재 점수와 최고 점수를 실시간으로 화면에 표시합니다.
- **Tile Rendering**: 그리드 상태 변화에 따라 타일의 위치와 숫자를 DOM에 동기화합니다.
- **Animations**:
    - **Movement Animation**: 타일이 이동할 때 부드럽게 미끄러지는 효과(CSS Transition).
    - **Merge Animation**: 타일이 합쳐질 때 크기가 커졌다가 돌아오는 팝 효과(Scale effect).
    - **New Tile Pop-up**: 새로운 타일 생성 시 나타나는 효과.
- **Game Over/Win Overlay**: 게임 종료 또는 승리 시 상태를 알리는 모달 창을 표시합니다.

### 4.3 입력 처리 (`main.js`)
- **Keyboard Input**: 키보드 방향키(Arrow Keys) 및 WASD 키 지원.
- **Touch Support**: 모바일 환경을 고려한 Swiping(스와이프) 제스처 인식 구현.

## 5. 구현 우선순위
1. **Phase 1: Core Engine 개발** (그리드 생성, 이동 로직, 병합 로직 완성)
2. **Phase 2: 기본 UI 구현** (HTML/CSS 구조 잡기, 점수 표시)
3. **Phase 3: 애니메이션 및 인터랙션 추가** (CSS Transition 적용, 스와이프 기능)
4. **Phase 4: 게임 상태 관리 및 예외 처리** (Win/Lose 판정, 재시작 기능)
5. **Phase 5: 최적화 및 폴리싱** (코드 리팩토링, 모바일 반응형 레이아웃 점검)
