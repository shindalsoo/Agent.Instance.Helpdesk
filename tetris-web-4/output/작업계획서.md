# [작업계획서] Web-based Tetris Game 개발 프로젝트

## 1. 프로젝트 개요
본 프로젝트는 HTML, CSS, JavaScript를 사용하여 웹 브라우저에서 실행 가능한 클래식 테트리스(Tetris) 게임을 구현하는 것을 목표로 합니다. 별도의 외부 라이브러리 없이 순수 자바스크립트(Vanilla JS)를 사용하여 가볍고 빠른 성능의 게임 엔진을 구축합니다.

## 2. 기술 스택
- **Language**: HTML5, CSS3, JavaScript (ES6+)
- **Rendering**: HTML5 Canvas API
- **Environment**: Web Browser (Chrome, Edge, Safari 등 최신 브라우저)

## 3. 파일 구조
모든 소스 코드는 `src/` 디arr 폴더 내에 관리됩니다.

```text
project-root/
├── index.html              # 게임 실행 메인 페이지
└── src/
    ├── constants.js        # 테트리미노 모양, 색상, 그리드 설정 등 상수 정의
    ├── style.css           # 게임 UI 및 레이아웃 스타일링
    ├── game.js             # 핵심 게임 로직 (엔진, 충돌 감지, 라인 클리어)
    └── utils.js            # 보조 함수 (랜덤 생성, 그리드 초기화 등)
```

## 4. 기능 명세

### 4.1. 게임 엔진 및 코어 로직
- **Grid System**: 10x20 크기의 플레이 필드 구현.
- **Tetromino Logic**:
    - 7가지 표준 테트리미노(I, J, L, O, S, T, Z) 구현.
    - 각 조각의 회전(Rotation) 알고리즘 적용 (Matrix rotation).
    - 중력(Gravity) 시스템: 일정 시간마다 조각이 아래로 이동.
- **Movement**:
    - 좌/우 이동 (Left/Right movement).
    - 소프트 드롭 (Soft Drop): 아래 방향키 입력 시 빠른 하강.
    - 하드 드롭 (Hard Drop): 스페이스바 입력 시 즉시 바닥으로 이동.

### 4.2. 충돌 및 규칙 시스템
- **Collision Detection**:
    - 벽면 충돌 감지.
    - 바닥 충돌 감지.
    - 이미 쌓인 블록과의 충돌 감지.
- **Line Clearing**: 한 줄이 가득 차면 해당 줄을 삭제하고 상단 블록들을 아래로 이동.
- **Scoring System**: 제거된 라인 수에 따른 점수 산정 (1줄, 2줄, 3줄, 4줄(Tetris) 각각 다른 점수 부여).

### 4.3. UI/UX 및 게임 상태 관리
- **Game Loop**: `requestAnimationFrame`을 이용한 부드러운 프레임 업데이트.
- **Game State**:
    - **Start/Pause**: 게임 시작 및 일시정지 기능.
    - **Game Over**: 블록이 천장에 닿았을 때 게임 종료 처리 및 재시작 버튼 활성화.
- **Display**:
    - 현재 점수(Score) 표시.
    - 다음 블록 미리보기(Next Piece) 기능 (선택 사항).

## 5. 구현 우선순위

1.  **Phase 1: 기초 환경 구축**
    - 프로젝트 폴더 구조 생성 및 HTML/CSS 기본 레이아웃 작성.
    - `constants.js`에 테트리미노 데이터 정의.
2.  **Phase 2: 핵심 엔진 개발 (Core Engine)**
    - Canvas 렌더링 시스템 구현.
    - 블록 생성 및 이동(좌, 우, 하) 로직 구현.
    - 회전(Rotation) 알고리즘 구현.
3.  **Phase 3: 게임 규칙 적용 (Game Rules)**
    - 충돌 감지(Collision) 로직 완성.
    - 라인 클리어(Line Clear) 및 점수 계산 시스템 구현.
    - 게임 오버(Game Over) 조건 처리.
4.  **Phase 4: UI/UX 고도화**
    - 점수판, 시작 버튼, 다음 블록 미리보기 등 UI 요소 완성.
    - 키보드 이벤트 리스너 최적화.
5.  **Phase 5: QA 및 폴리싱**
    - 버그 수정 (회전 시 벽 끼임 현상 등).
    - 애니메이션 효과 추가 및 코드 리팩토링.
