# Tetris – Simple Web‑Browser Game

## Project Overview  
A classic Tetris clone that runs entirely in the browser using only **HTML, CSS, and vanilla JavaScript**.  
The game is playable with keyboard arrows (← → ↑ ↓) and has a responsive layout so it works on desktop and mobile browsers.

---

## Technology Stack  

| Layer | Tech | Reason |
|-------|------|--------|
| UI | HTML5 | Semantic markup for the canvas, score panel, and controls. |
| Styling | CSS3 (Flexbox + Grid) | Simple, responsive design; no external frameworks. |
| Logic | JavaScript ES6+ | Game loop, rendering, collision detection, input handling. |

No build tools or libraries are required – the game can be opened directly in a browser.

## Feature Specification  

| Feature | Description | Implementation Notes |
|---------|-------------|----------------------|
| **Game Board** | 10 columns × 20 rows grid. Each cell holds a color or empty state. | Represented as a 2‑D array (`board[row][col]`). |
| **Tetrominoes** | Seven standard shapes (I, J, L, O, S, T, Z). | Stored in `piece.js` as arrays of rotation states; each state is an array of coordinate offsets. |
| **Piece Movement** | Left/Right shift, soft drop (down), hard drop (space or up arrow). | Update piece position on key events; check collision before moving. |
| **Rotation** | 90° clockwise rotation using matrix transform or pre‑computed states. | Use the stored rotation arrays for simplicity. |
| **Collision Detection** | Prevent pieces from overlapping existing blocks or going out of bounds. | Before each move/rotation, test target positions against board state and boundaries. |
| **Line Clearing** | Detect full rows, remove them, shift above rows down, update score. | Score: 40 points for first line, double per additional simultaneous line (classic Tetris). |
| **Game Loop** | Automatic descent timer that speeds up as level increases. | `requestAnimationFrame` + timestamp diff to control tick rate. |
| **Controls** | Arrow keys for movement/rotation; space for hard drop; 'P' to pause/resume. | Add event listeners on `window`. |
| **Display** | Score, level, next piece preview, and lines cleared shown beside the board. | Use simple `<div>` elements styled with CSS Grid/Flexbox. |
| **Responsive Design** | Canvas scales to fit screen width while maintaining cell size ratio. | Calculate cell size based on container width; keep aspect ratio 10:20. |
| **Game Over** | When a new piece cannot spawn, show overlay "Game Over" and reset option. | Overlay div with `position:absolute` covering the canvas. |
| **Sound (Optional)** | Simple click or beep for line clears / rotations. | Use Web Audio API or simple `<audio>` tags. |

## Implementation Priorities  

1. **Board & Rendering** – Create the grid, draw cells in Canvas.
2. **Tetromino Definition** – Load shapes and rotation states.
3. **Piece Movement + Collision** – Basic left/right/down movement with collision checks.
4. **Rotation Logic** – Implement clockwise rotation with bounds check.
5. **Line Clearing & Scoring** – Detect full rows, clear them, update score.
6. **Game Loop & Timing** – Automatic descent and speed scaling.
7. **Input Handling** – Keyboard controls for all actions.
8. **UI Elements** – Score/level display, next piece preview.
9. **Responsive Layout** – Scale canvas to fit different screen sizes.
10. **Extras** – Pause/resume, game‑over overlay, optional sound.

## Development Notes  

- **Canvas Size Calculation:**  
  ```js
  const cellSize = Math.floor(container.clientWidth / BOARD_COLS);
  canvas.width = cellSize * BOARD_COLS;
  canvas.height = cellSize * BOARD_ROWS;
  ```
- **Game Loop Skeleton:**
  ```js
  let lastTime = 0;
  function gameLoop(time) {
      if (time - lastTime > tickDuration) {
          update();   // move piece down
          render();
          lastTime = time;
      }
      requestAnimationFrame(gameLoop);
  }
  ```
- **Collision Function Signature:**
  ```js
  function canMove(piece, offsetX, offsetY) : boolean
  ```
