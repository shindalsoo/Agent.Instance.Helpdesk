# 작업계획서: 웹 테트리스 게임 개발

## 1. 프로젝트 개요
본 프로젝트는 별도의 설치 없이 웹 브라우저에서 즉시 실행 가능한 클래식 테트리스(Tetris) 게임을 개발하는 것을 목표로 합니다. HTML5 Canvas API를 사용하여 그래픽을 렌더링하며, 순수 JavaScript(Vanilla JS)를 통해 게임 로직을 구현합니다.

- **목표**: 사용자에게 직관적이고 재미있는 테트리스 플레이 경험 제공
- **주요 특징**: 실시간 조작, 라인 클리어 점수 시스템, 게임 오버 처리, 반응형 레이아웃

## 2. 기술 스택
- **Language**: HTML5, CSS3, JavaScript (ES6+)
- **Rendering**: HTML5 Canvas API
- **Environment**: 웹 브라우저 (Chrome, Edge, Safari 등 최신 브라우저)

## 3. 파일 구조
모든 소스 코드는 `src/` 폴더 내에 위치합니다.

```text
project_root/
│
├── index.html          # 게임 실행 메인 페이지
├── style.css           # 게임 UI 및 레이아웃 스타일링
├── src/
│   ├── main.js         # 게임 루프 및 초기화 로직
│   ├── tetromino.js    # 테트리미노(블록) 데이터 및 동작 로직
│   ├── board.js        # 게임 보드(Grid) 관리 및 충돌 감지 로직
│   └── renderer.js     # Canvas 드로잉 및 UI 업데이트 로직
└── output/
    └── 작업계획서.md    # 본 문서
```

## 4. 기능 명세

### 4.1 게임 엔진 (Core Logic)
- **Game Loop**: `requestAnimationFrame`을 사용하여 일정한 프레임 레이트로 화면을 갱신합니다.
- **Gravity/Tick System**: 일정 시간 간격마다 테트리미노를 아래로 한 칸씩 이동시킵니다.
- **Collision Detection**: 블록이 벽, 바닥 또는 다른 블록과 충돌했는지 판별하여 이동 및 회전을 제한합니다.

### 4.2 테트리미노 (Tetromino) 관리
- **Block Types**: I, J, L, O, S, T, Z 형태의 7가지 블록 구현.
- **Rotation**: 각 블록은 90도 단위로 회전 가능해야 하며, 벽 충돌 시 'Wall Kick' 최소 로직 적용.
- **Spawning**: 새로운 블록이 상단에 생성될 때 보드 상태를 확인하여 게임 오버 여부를 결정합니다.

### 4.3 게임 규칙 및 시스템
- **Line Clearing**: 한 줄이 가득 차면 해당 라인을 삭제하고, 상단의 모든 블록을 아래로 내립니다.
- **Scoring System**: 클리어한 라인 수에 따라 점수를 누적 계산합니다. (1줄 < 2줄 < 3줄 < 4줄 순으로 가산점 부여)
- **Game Over**: 새로운 블록이 생성될 위치가 이미 차 있는 경우 게임을 종료하고 'Game Over' 메시지를 표시합니다.

### 4.4 사용자 인터페이스 (UI/UX)
- **Controls**: 키보드 방향키(좌, 우, 하) 및 상단 방향키(회전)를 통한 조작 지원.
- **Score Display**: 현재 점수를 실시간으로 화면에 표시.
- **Visual Feedback**: 라인 클리어 시 시각적 효과 또는 블록 색상 구분을 통해 가독성 확보.

## 5. 구현 우선순위
1.  **P0 (Critical)**: HTML Canvas 설정 및 테트리미노 기본 드로잉, 중력 로직(하강).
2.  **P0 (Critical)**: 충돌 감지 시스템(벽, 바닥, 다른 블록) 및 라인 클리어 구현.
3.  **P1 (High)**: 키보드 입력 처리 및 테트리미노 회전 로직.
4.  **P1 (High)**: 점수 계산 및 게임 오버 상태 관리.
5.  **P2 (Medium)**: CSS를 활용한 UI 디자인(점수판, 다음 블록 표시 등) 개선.
