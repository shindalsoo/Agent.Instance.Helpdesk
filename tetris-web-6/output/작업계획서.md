# Tetris Web Game Development Plan

## 1. Project Overview
The goal of this project is to develop a fully functional, web-based Tetris game. The game will feature classic Tetris mechanics including tetromino movement, rotation, line clearing, scoring, and increasing difficulty levels. The development follows a structured pipeline: Requirement Analysis (Planning), Implementation (Development), and Quality Assurance (Review).

## 2. Tech Stack
- **Language**: HTML5, CSS3, JavaScript (ES6+)
- **Rendering**: HTML5 Canvas API
- **Environment**: Web Browser (Chrome, Firefox, Safari, Edge)

## 4. Feature Specification

### 4.1 Core Game Engine (`engine.js`)
- **Grid System**: A 10x20 matrix representing the game board.
- **Gravity Logic**: Automatic downward movement of the active tetromino at regular intervals.
- **Collision Detection**: Logic to prevent pieces from moving through walls, floor, or existing settled pieces.
- **Game Loop**: Using `requestAnimationFrame` for smooth rendering and timing control.

### 4.2 Movement & Rotation (`tetrominoes.js`, `main.js`)
- **Controls**: 
    - Left/Right Arrow: Horizontal movement.
    - Down Arrow: Soft drop (accelerated gravity).
    - Up Arrow / Rotate Key: Rotate the piece around its center.
    - Space Bar: Hard drop (instant placement).
- **Rotation Logic**: Implementation of Wall Kicks (basic) to allow rotation even when near edges.

### 4.3 Scoring & Progression (`engine.js`)
- **Line Clearing**: Detect full rows, remove them, and shift upper rows down.
- **Scoring System**:
    - 1 Line: 100 pts
    - 2 Lines: 300 pts
    - 3 Lines: 500 pts
    - 4 Lines (Tetris): 800 pts
- **Leveling**: Increase gravity speed every time the player reaches a certain score threshold.

### 4.4 UI/UX (`ui.js`, `style.css`)
- **Game Canvas**: The main visual area for the active and settled pieces.
- **Side Panel**:
    - **Next Piece Preview**: A small canvas showing the upcoming tetromino.
    - **Scoreboard**: Real-time display of current score and level.
- **Game Over Overlay**: An overlay screen that appears when a collision occurs at the top, providing a "Restart" button.

## 5. Implementation Priority

1.  **Phase 1: Foundation (Setup)**
    - Setup project structure and HTML/CSS scaffolding.
    - Define constants (colors, shapes, grid size).
2.  **Phase 2: Core Mechanics (Development - Level 1)**
    - Implement the grid and basic tetromino rendering on Canvas.
    - Implement gravity and basic downward movement.
3.  **Phase 3: Interaction (Development - Level 2)**
    - Implement left/right/down movement.
    - Implement rotation logic and collision detection.
4.  **Phase 4: Game Rules (Development - Level 3)**
    - Implement line clearing and scoring.
    - Implement "Next Piece" preview and level progression.
5.  **Phase 5: Polish & QA (Review)**
    - Add Game Over state and Restart functionality.
    - UI/UX refinement (CSS animations, layout).
    - Final bug testing (collision edge cases, rotation glitches).
