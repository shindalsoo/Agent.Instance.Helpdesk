# 작업계획서: Web-based Tetris Game 개발

## 1. 프로젝트 개요
본 프로젝트는 HTML5, CSS3, Vanilla JavaScript를 사용하여 웹 브라우저에서 실행 가능한 클래식 테트리스(Tetris) 게임을 개발하는 것을 목표로 합니다. 별도의 외부 라이브러리 없이 순수 자바스크립트만으로 구현하여 가볍고 빠른 성능을 보장하며, 반응형 디자인을 통해 데스크톱과 모바일 환경 모두에서 플레이할 수 있도록 합니다.

## 2. 기술 스택
- **Language**: HTML5, CSS3, JavaScript (ES6+)
- **Rendering**: HTML5 Canvas API
- **Environment**: Modern Web Browsical (Chrome, Firefox, Safari, Edge)

## 3. 파일 구조
모든 소스 코드는 `src/` 디렉토리 내에 위치합니다.

```text
project_root/
│
└── src/
    ├── index.html          # 게임 메인 엔트리 포인트 및 UI 구조
    ├── css/
    │   └── style.css       # 게임 보드, UI 요소 및 반응형 레이아웃 스타일링
    └── js/
        ├── constants.js    # 테트로미노 모양, 색상, 그리드 크기 등 상수 정의
        ├── game.js         # 핵심 게임 엔진 (루프, 충돌 감지, 로직)
        ├── input.js        # 키보드 및 터치 이벤트 핸들러
        ├── ui.js           # 점수 업데이트, 다음 블록 표시 등 DOM 조작
        └── utils.js        # 보조 함수 (랜덤 생성, 그리드 복사 등)
```

## 4. 기능 명세

### 4.1 핵심 게임 엔진 (Core Engine)
- **Grid System**: 10x20 크기의 플레이 영역(Grid)을 관리합니다.
- **Tetromino Generation**: 7가지 표준 테트로미노(I, J, L, O, S, T, Z)를 무작위로 생성합니다.
- **Game Loop**: `requestAnimationFrame`을 사용하여 부드러한 프레임 드랍 없는 애니메이션을 구현하고, 일정 시간 간격으로 중력(Gravity)에 의한 자동 하강을 처리합니다.
- **Collision Detection**: 블록이 벽, 바닥 또는 이미 배치된 다른 블록과 충돌하는지 체크합니다.

### 4.2 조작 및 물리 (Movement & Physics)
- **Horizontal Movement**: 좌/우 방향키를 통한 이동.
- **Rotation**: 상단 방향키 또는 특정 키를 이용한 블록 회전 (벽 차기/Wall Kick 로직 포함 권장).
- **Soft Drop**: 아래 방향키를 통한 빠른 하강.
- **Hard Drop**: 스페이스바를 이용해 즉시 바닥으로 낙하 및 고정.

### 4.3 점수 및 라인 시스템 (Scoring & Line Clearing)
- **Line Clear**: 한 줄이 블록으로 가득 차면 해당 줄을 삭제하고 상단 블록들을 아래로 이동시킵니다.
- **Scoring System**: 제거된 라인 수(1줄, 2줄, 3줄, Tetris)에 따라 가중치를 부여하여 점수를 계산합니다.
- **Level Up**: 누적 점수에 따라 게임 속도(Gravity speed)가 점진적으로 증가하는 난이도 조절 기능을 포함합니다.

### 4.4 UI/UX 및 인터페이스 (UI/UX)
- **Score Display**: 현재 점수, 최고 점수를 화면에 실시간 표시합니다.
- **Next Piece Preview**: 다음에 나올 블록을 별도의 작은 캔버스에 미리 보여줍니다.
- **Game Over Screen**: 게임 종료 시 결과 메시지와 '다시 시작' 버튼을 제공합니다.
- **Responsive Design**: CSS Media Queries를 사용하여 모바일 기기에서도 게임 보드가 화면에 적절히 보이도록 조정하며, 터치 컨트롤러(좌/우/회전 버튼)를 구현합니다.

## 5. 구현 우선순위

1.  **Phase 1: 기초 환경 구축**
    - `src/index.html` 및 기본 CSS 레이아웃 설정.
    - `constants.js`에 테트로미노 데이터 구조 정의.
2.  **Phase 2: 핵심 로직 구현 (Core Engine)**
    - Canvas 기반 그리드 렌더링.
    - 블록 생성 및 하강(Gravity) 로직 구현.
    - 충돌 감지(Collision Detection) 알고리즘 완성.
3.  **Phase 3: 상호작용 구현 (Input & Rotation)**
    - 키보드 이벤트 리스너 연동.
    - 블록 회전(Rotation) 및 벽 충돌 처리 로직 개발.
4.  **Phase 4: 게임 규칙 완성 (Scoring & Line Clear)**
    - 라인 삭제 및 점수 계산 시스템 구현.
    - 레벨업에 따른 속도 변화 로직 추가.
5.  **Phase 5: UI/UX 고도화 및 반응형 대응**
    - Next Piece 프리뷰 기능.
    - 모바일용 터치 버튼 인터페이스 추가.
    - 게임 종료(Game Over) 처리 및 재시작 기능.
6.  **Phase 6: QA 및 최적화**
    - 버그 수정 (회전 시 벽 끼임 현상 등).
    - 코드 리팩토링 및 성능 최적화.
